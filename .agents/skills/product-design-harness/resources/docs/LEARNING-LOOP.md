# Learning Loop

UX3 learning converts product feedback and failure into stronger judgment. The loop keeps machine evidence, human interpretation, and accountable choice separate.

```text
signal -> interpretation -> counter-signal -> decision impact
       -> human judgment -> bounded action -> observed outcome
       -> evidence update or rule update
```

## Steps

| Step | Required separation |
|---|---|
| Signal | Record what was observed before explaining it. |
| Interpretation | State what the reviewer thinks the signal means. |
| Counter-signal | Name what weakens, contradicts, or reframes the interpretation. |
| Decision impact | State what changes because of the evidence. |
| Human judgment | Record taste, ethics, meaning, strategy, or risk appetite as accountable human input. |
| Bounded action | Limit the next action by evidence tier, risk, reversibility, and recovery capacity. |
| Observed outcome | Capture what happened after action. |
| Evidence update or rule update | Update the evidence record, add a test or evaluation, create a rule, or record an accepted risk. |

## Failure Becomes System Memory

A repeated failure is not complete learning until it becomes one of:

| Artifact | Use |
|---|---|
| Reproducible case | The failure can be observed again. |
| Test | The failure is prevented by automation. |
| Evaluation | The failure is measured in a recurring review. |
| Decision rule | The failure changes future product judgment. |
| Accepted risk | A named human records why the risk is tolerated and when to revisit it. |

## Stop Conditions

Stop or reframe when:

- source, signal, interpretation, or decision impact is missing;
- counter-signals have not been inspected for a material decision;
- human-owned judgment is hidden inside evidence language;
- the next action is larger than the current evidence can justify;
- a repeated failure has no test, evaluation, rule, or accepted-risk record.

The loop is complete only when the observed outcome changes evidence, changes a rule, or explicitly preserves the current rule with a recorded reason.
