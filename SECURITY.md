# Security Policy

## Supported Version

Security fixes are applied to the current `main` branch. The repository does
not provide security support for forks, modified copies, or generated product
decisions after they leave this repository.

## Report A Vulnerability

Use GitHub's private vulnerability reporting for this repository. Do not open
a public issue for a suspected vulnerability, leaked credential, prompt
injection path, unsafe tool instruction, schema bypass, or supply-chain issue.

Include:

- the affected file, command, or workflow;
- the smallest reproducible example;
- the potential impact;
- whether the issue can expose data or authorize an external action; and
- a suggested mitigation, when available.

The maintainers will acknowledge a valid report, assess severity, and publish a
fix or documented boundary. A schema-valid review is not proof that its source
evidence is true; evidence authenticity remains an explicit trust boundary.

## Scope

Reports may cover the Skill entrypoint, prompts, agent definitions, schemas,
validators, examples, installer instructions, website, and CI workflows.
